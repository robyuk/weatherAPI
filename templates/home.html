Weather Data API

URL Format: http://127.0.0.1:5001/api/v1/STAID/DATE

Where STAID is the station ID in the table below
and DATE is optional and in the format YYYY[MM[DD]]

URL Example: http://127.0.0.1:5001/api/v1/10/19881025
to get the average temperature recorded at the Stockholm station on 25th October 1988.
This will return a 'status'='OK' if the STAID and DATE are found, and dict of dicts as follows:

{{example|safe}}

Where the 'location' dict keys are:
STAID : Station identifier
STANAME: Station name
CN : Country code (ISO3116 countrycodes)
LAT : Latitude in degrees:minutes:seconds (+: North, -: South)
LON : Longitude in degrees:minutes:seconds (+: East, -: West)
HGTH : Station elevation in meters

And the 'data' dict keys are:
SOUID: Source identifier
Date : Date YYYY-MM-DD
Celsius : Daily mean temperature in 'C
Fahrenheit : Daily mean temperature in 'F
Q_TG : quality code for temperature data (0='valid'; 1='suspect'; 9='missing')

If the STAID is not found, only {'status': 'Station not found'} is returned.
If the 'DATE' is invalid, {'status': 'Invalid date'} is returned. This may indicate an invalid date format or date out of range for the given station

URL Example: http://127.0.0.1:5001/api/v1/10/198810
to get the average temperature recorded at the Stockholm station on every day during the month of October 1988. In this case a dict of 'location' (as above) and 'dataframe' dict containing 'data' dicts by index starting at zero.

URL Example: http://127.0.0.1:5001/api/v1/10/1988
to get the average temperature recorded at the Stockholm station on every day of 1988.

URL Example: http://127.0.0.1:5001/api/v1/10
to get the average temperature recorded at the Stockholm station for all dates.
